# 002 API

## Purpose
Define the external API contracts consumed by this plugin (Mapflow backend REST API).

## Content

Interface type: REST JSON over HTTPS (plugin is the client, Mapflow backend is the server).

Base URL: `https://whitemaps-{env}.mapflow.ai/rest` where `{env}` is one of: production (empty), staging, internal, duty.

### Authentication

Two mechanisms:
1. **Basic Auth (Token):** Header `Authorization: Basic {base64(token:)}`. Token stored in `QgsSettings("mapflow/token")`.
2. **OAuth2 (Keycloak):** Authorization Code with PKCE. Realm at `https://auth-{env}.mapflow.ai/auth/realms/mapflow-{env}/`. Tokens managed by QGIS Auth Manager.

Custom header on all requests: `x-plugin-version: {version}`.

### Endpoints consumed

**Projects:**
- `POST /projects` — create project
- `GET /projects/{id}` — get project (includes billingType, workflowDefs, shareProject)
- `POST /projects/page` — list projects with pagination
- `PUT /projects/{id}` — update project
- `DELETE /projects/{id}` — delete project

**Processings:**
- `POST /projects/{project_id}/processings` — submit processing job (geometry + params)
- `GET /projects/{project_id}/processings/v2` — list processings for project (polled every ~5s)
- `POST /projects/{project_id}/processings/v2/page` — list processings with pagination, sorting & filtering
- `PUT /processings/{id}` — update processing name/description

**Data Catalog (My Imagery):**
- `POST /rasters/mosaic` — create mosaic (imagery collection)
- `GET /rasters/mosaic` — list mosaics
- `GET /rasters/mosaic/{id}` — get mosaic details
- `PUT /rasters/mosaic/{id}` — update mosaic
- `DELETE /rasters/mosaic/{id}` — delete mosaic
- `POST /rasters/mosaic/{id}/image` — upload image to mosaic (multipart, 1h timeout)
- `POST /rasters/mosaic/image` — create mosaic + upload first image
- `GET /rasters/mosaic/{id}/image` — list images in mosaic
- `GET /rasters/image/{id}` — get image details
- `PUT /rasters/image/{id}` — update image (rename)
- `DELETE /rasters/image/{id}` — delete image
- `GET /rasters/memory` — user storage limits

**Imagery Search:**
- `GET /catalog/meta` — search imagery across providers (provider + filters)
- `GET /meta/skywatch/id` — Sentinel-2 metadata

### Error model

Multiple formats depending on source:
- Standard API: `{"code": string, "message": string, "params": object}`
- Data Catalog: `{"detail": {error_data}}`
- Fallback: raw response text

### External APIs (non-Mapflow)

- **Maxar SecureWatch WFS:** `https://securewatch.digitalglobe.com/catalogservice/wfsaccess` — XML WFS GetFeature for image metadata search.
- **Maxar WMTS:** `https://securewatch.digitalglobe.com/earthservice/wmtsaccess` — tile service for image preview.
- **Sentinel-2 Preview:** `https://preview.skywatch.com/esa/sentinel-2/{id}.jp2` — JPEG2000 preview images.

### Versioning policy
The plugin declares `x-plugin-version` on every request. The backend may respond with update notifications. Breaking API changes are reflected in major plugin version bumps.
