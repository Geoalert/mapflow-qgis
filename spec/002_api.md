# 002 API

## Purpose
Define the external API contracts consumed by this plugin (Mapflow backend REST API).

## Content

Interface type: REST JSON over HTTPS (plugin is the client, Mapflow backend is the server).

Base URL: `https://whitemaps-{env}.mapflow.ai/rest` where `{env}` is one of: production (empty), staging, internal, duty.

### Authentication

Two mechanisms:
1. **Basic Auth (Token):** Header `Authorization: Basic {base64(token:)}`. Token stored in `QgsSettings("mapflow/token")`.
2. **OAuth2 (Keycloak):** Authorization Code with PKCE. Realm at `https://auth-{env}.mapflow.ai/auth/realms/mapflow-{env}/`. Tokens managed by QGIS Auth Manager.

Custom header on all requests: `x-plugin-version: {version}`.

### Endpoints index

Detailed endpoint documentation is split into sub-files:

- **[002_A_project_api.md](002_A_project_api.md)** — Projects: CRUD, pagination, sharing
- **[002_B_processing_api.md](002_B_processing_api.md)** — Processings: submit, list, update
- **[002_C_myimagery_api.md](002_C_myimagery_api.md)** — Data Catalog (My Imagery): mosaics, images, upload, download, storage
- **[002_D_search_api.md](002_D_search_api.md)** — Imagery Search: catalog search, external APIs (Maxar, Sentinel)
- **[002_F_plan_processing_api.md](002_F_plan_processing_api.md)** — Planned processing templates: CRUD, run, status, and template-linked processings

### Error model

Multiple formats depending on source:
- Standard API: `{"code": string, "message": string, "params": object}`
- Data Catalog: `{"detail": {error_data}}`
- Fallback: raw response text

### Versioning policy
The plugin declares `x-plugin-version` on every request. The backend may respond with update notifications. Breaking API changes are reflected in major plugin version bumps.
